## 1. Bugs JS purs & dead code

### Bloquants
- **L6452** — `currentUser.name.split(' ')` sans null-check → crash sidebar au démarrage si Google n'a pas fini l'init.
- **L1369 / L1749** — `filtreProprioId` utilisé avant sa déclaration (TDZ). Si `renderHistoriqueDocs()` se déclenche avant navigation, ReferenceError.
- **L4794** — Typo : `${(b.type||'')=== t?'selected':''}` (triple `=` parse comme affectation). Le select Type d'un bien existant n'a jamais l'option pré-sélectionnée à l'ouverture en édition.
- **L6250** — `_originalSaveData()` appelée dans `firestoreLoadOrCreate()` mais définie L6636 : au premier merge Firestore, "not a function".

### Majeurs
- **L6213** — `await cred.user.updateProfile({ displayName })` dans `try { … } catch(e) {}` sans log. Échec silencieux de l'inscription email → sidebar "undefined".
- **L6457-6463** — `updateSidebarUser()` lit `currentUser.picture` puis `currentUser.name` sans revérifier : si le second est absent, plantage après avoir partiellement rendu.
- **L7782** — `eq.groupedEcheances.forEach()` : `groupedEcheances` peut être `undefined` pour les relances non groupées. Crash à l'envoi.
- **L2647 / L3688** — `window._toggleTimers[key] = setTimeout(...)` sans init `window._toggleTimers = {}`. Fonctionne par accident grâce à la coercion, mais fragile.

### Mineurs / dead code
- **L2555** — `ficheBailPerioChange()` définie mais jamais appelée.
- **L7885** — `setInterval(_gisCheck)` sans `clearInterval` : petite fuite à chaque reload SPA.
- **L5779, L6465** — Catchs qui avalent l'erreur (`console.warn` ou `{}`) : échecs de backup et de bannière délégué invisibles.
- **L6109** — `fbAuth.setPersistence(...).catch(console.error)` non-awaited, session peut ne pas être persistée.

### Incohérences de data model
- Champ `locataireId` dans `historique` qui en vrai vaut `bail.locId` (alias confus).
- Pas de validation `loyer > 0` (juste `Math.max(0, ...)`) → bail à 0€ possible.

## 2. UI / UX / Thème

### Bloquants — dark mode cassé
- **L95** — `.sidebar { background: #FFFFFF }` sans override `body.dark-mode` : sidebar blanche sur fond noir.
- **L143** — Même problème sur `.topbar`.
- **L181, L186** — Inputs avec `background: rgba(255,255,255,0.7/0.85)` en dur : pas de feedback focus en dark mode.
- **L855, L1022, L1403, L2204, L7511, L7522** — `style="background:white"` inline sur selects/inputs : champs invisibles en dark mode (texte clair sur fond clair forcé).
- **L1116, L2511, L2912** — Boutons "Archiver sur Drive" en `#1A1A2E` sur fond dark → disparaissent.

### Majeurs — unicode en dur dans HTML statique
Les séquences `\u00XX` ne sont interprétées que dans les **chaînes JS**. Dans un attribut `title=""` ou entre tags HTML, elles s'affichent littéralement.
- **L2181** — `title="Modifier le montant du d\u00e9p\u00f4t"` → affiche `d\u00e9p\u00f4t`.
- **L2268, L2276, L2375, L2382** — Tooltips des cartes rendement : `march\u00e9 estim\u00e9e`.
- **L3312, L3394, L3430** — Icônes `\u270E` dans les boutons "Modifier".
- Des chaînes dynamiques (ex. `'Financi\u00e8re'`, `'Soci\u00e9t\u00e9'`) stockées dans les données : à vérifier côté loadData / import JSON.

### Majeurs — contrastes & accessibilité
- Empty states en `var(--text-light)` (#6B6D82) sur `#F7F7F8` : contraste borderline WCAG AA.
- Inputs dans le login (L748-753) sans `<label for="">` associé.
- Emoji 🌙 du toggle dark mode sans texte accessible ni aria-label explicite de l'état courant.
- Alertes en `#fff3cd` / `#ffc107` sans override dark → jaune pâle illisible.
- Bordures de tableau `#F5F5F7` qui disparaissent en dark mode.

### Mineurs — incohérences de palette
- 3 styles différents pour le même bouton "Archive Drive" (gradient vert ailleurs, bleu sombre ici, violet là).
- Mix "Suppr." / "Supprimer" selon les écrans.
- Modal headers : email en `#6C5CE7` fixe, confirmation en couleur thème.
- 6 breakpoints responsive différents (360, 480, 768…) sans système unifié.

## 3. Logique métier

### Bloquants
- **Rendement locatif — doublon charges (L5837)**
  `loyerAppele` inclut déjà `chargesAppelees`, puis le code fait `loyerAnTotal += chargesBailAn` après la boucle. Les rendements sortent surestimés (~+3 % à +4 % selon le bail).
- **Rendement à l'achat = 0 si coût = 0 (L5865)**
  Pour un bien hérité / sans prix d'achat, la plus-value n'est pas "0" mais "N/A". Affichage trompeur.

### Majeurs
- **Quittance PDF sans vérifier le paiement (L2980, L5192-5306)**
  `genererDocuments()` ne regarde jamais `getPaiementStatut()`. Possible génération d'une quittance (= paiement reçu) pour un mois impayé → document juridiquement faux.
- **Combiné "Quittance + Avis" — mois précédent mal calculé (L2987-2994)**
  Logique `prevIdx` ambiguë sur les baux trimestriels si le mois demandé n'est pas un mois de trimestre. Génère des docs avec montants à 0 ou mois décalés.
- **Prorata pas appliqué au rendement annuel (L5829)**
  Un bail commencé le 15 mai affiche l'année courante sur 12 mois "pleins". Première et dernière année d'un bail biaisées.
- **Rendement NET ignore les provisions locataire (L4532, L5842)**
  `getChargesAnnuelles` ne déduit que les charges propriétaire. Net = trop optimiste.
- **Statut "Résilié" — impayés antérieurs disparaissent silencieusement (L7081)**
  `collectImpayes()` filtre `statut === 'Actif'`. Aucun garde-fou avant de passer un bail en Résilié s'il reste des créances.
- **Affichage société vs particulier dans PDFs & emails (L5200, L5315, L5422)**
  Templates utilisent `${loc.civilite} ${loc.prenom} ${loc.nom}` sans helper. Pour une société, on peut obtenir "Société Houssem Manallah MH Transport" selon la saisie.
- **Relance email silencieusement sautée si pas d'email (L7381)**
  Alert puis rien. L'utilisateur ne sait pas si l'action a été appliquée aux autres échéances.

### Mineurs
- `bail.fin = ''` vs `null` : pas de bug immédiat mais comparaisons futures risquées.
- Champ `type` + `civilite` redondants pour les sociétés (`civilite === 'Société'` vs `type === 'Société'`).
- Pas de statut de paiement **partiel** (binaire payé/impayé uniquement).
- `documentsGeneres` plafonné à 1000 entrées, silencieusement tronqué.
- Historique n'identifie pas l'utilisateur en mode délégué (pas de "qui").

## 4. Auth dual (Google + Firebase)

### Bloquants
- **Race condition Firebase (L6640)**
  Au 1er `saveData()` après inscription email, si `fbAuth.currentUser` n'a pas encore propagé, la branche Firestore est skippée. La sauvegarde retombe sur Drive/local → données silencieusement perdues.
- **Logout partiel → login cassé au F5 suivant (L6485-6486)**
  Si `localStorage.removeItem('gl_user')` échoue (quota), `gl_user` persiste. Au prochain F5 : `currentUser` restauré mais `fbAuth.currentUser` null → ni Firestore, ni Drive, écran à moitié connecté.

### Majeurs
- **Mode délégué — Drive écrase en multi-user (L1247-1252, L6643)**
  `getStorageKey()` utilise `currentUser.email`, pas `getDataOwner()`. Deux utilisateurs accédant aux données de Kevin peuvent s'écraser mutuellement au prochain sync Drive.
- **Token Drive expiré silencieusement (L6433)**
  `silentTokenRefresh` ne distingue pas "token expiré" et "app révoquée côté Google". Status bar dit "Sauvegarde…" puis rien.
- **Logout double → Firebase pas signOut (L6479)**
  `wasProvider` calculé après nullification, deuxième appel à logout laisse les refresh tokens Firebase en place.

### Mineurs
- `name: displayName || email.split('@')[0]` : sidebar "undefined" possible si les deux sont vides.
- `JSON.parse(gl_user)` sans try/catch → corruption localStorage ramène l'écran de login sans message.
- Pas de fallback explicite si les règles Firestore rejettent (erreur, mais pas surfacée).

## 5. User journeys — ruptures détectées

| Journey | État | Principales ruptures |
|---|---|---|
| **1. Nouveau user email** | ⚠️ | Badges pas refresh immédiat selon la page active ; F5 pendant saisie → données non sauvegardées (debounce 2 s) |
| **2. User Google F5** | ⚠️ | `gdriveLoadOrCreate` silencieux si 0 fichier Drive → reste sur données locales vides ; mode délégué pas signalé |
| **3. Cocher un paiement** | ✅ | Fonctionne ; KPI re-rendered via `requestAnimationFrame` (lag ~100 ms) |
| **4. Bail trimestriel** | ❌ | Grille affiche des mois à 0 € ambigus ; `collectImpayes` peut compter des mois sans charges comme impayés fantômes |
| **5. Relance groupée** | ⚠️ | Un seul email OK, CC OK — mais pas de bouton "Confirmer" explicite avant `gmail.send` ; quota Gmail silencieux |
| **6. Batch 10 quittances** | ❌ | Génère aussi pour les impayés ; pas de progress UI ; pas de distinction société/particulier dans le nom |
| **7. Édition dépôt** | ⚠️ | Deux chemins (modal + icône stylo) ne produisent pas le même log historique |
| **8. Résiliation** | ❌ | Les impayés antérieurs disparaissent des listes sans alerte ; paiements du bail résilié restent en base sans recalcul |
| **9. Export / import** | ❌ | Import n'utilise pas `_version` → régression possible ; pas de validation de schéma ; `gl_user` restauré avant import = conflit d'identité |
| **10. Deux onglets** | ⚠️ | Event `storage` détecte local seulement, sync Drive arrive 2-4 s plus tard sans notifier l'autre onglet ; dernier à écrire gagne |

## 6. Plan de correctifs proposé (ordre recommandé)

### Sprint 1 — bloquants (30-60 min)
1. Fixer le doublon `loyerAnTotal += chargesBailAn` (L5837).
2. Corriger le typo `=== t?` → `=== t ?` (L4794).
3. Remonter la déclaration de `_originalSaveData` / `filtreProprioId` / déclarer `window._toggleTimers = {}`.
4. Ajouter null-checks sur `currentUser.name` et `fbAuth.currentUser` avant écriture Firestore.
5. Ajouter override `body.dark-mode .sidebar`, `.topbar`, `.sidebar-btn` et variables pour inputs blancs.

### Sprint 2 — majeurs métier (1-2 h)
6. `genererDocuments()` : skip quittance si paiement non encaissé, ou forcer le type "Avis d'échéance" dans ce cas.
7. Prorata rendement annuel pour baux partiels.
8. Helper `getLocataireDisplayName(loc)` utilisé partout (quittances, emails, tableaux).
9. Alerte avant passage en "Résilié" s'il reste des impayés.
10. Déduire les provisions locataire du rendement NET (ou documenter clairement quel "net" est calculé).

### Sprint 3 — UI cleanup (1 h)
11. Grep & remplacer tous les `\u00XX` en HTML statique par les vrais caractères.
12. Unifier le style du bouton "Archive Drive".
13. Nettoyer les 6 breakpoints responsive → palette unifiée (mobile ≤640, tablet ≤1024).
14. Ajouter labels `for/id` sur les inputs de login.

### Sprint 4 — auth & multi-tab (1-2 h)
15. `getStorageKey()` basé sur `getDataOwner()` pour le mode délégué.
16. Queue `saveData()` tant que `fbAuth.currentUser` pas prêt.
17. Gestion explicite du refresh token Drive + feedback statusbar.
18. `storage` event → aussi déclencher un `loadData()` via Drive après 3 s pour rattraper les modifs distantes.
